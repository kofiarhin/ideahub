## 2026-07-23 — Separate communication, history, and authority

- **Context:** Zoro and Architect operational-memory implementation
- **Learning:** A reliable multi-agent loop needs three distinct durable layers: inbox files for communication and feedback, operational logs for chronological supporting history, and Architect run files for authoritative task and verification state.
- **Evidence:** `AGENT_COORDINATION.md`, `zoro/INSTRUCTIONS.md`, `architect/README.md`, `architect/commands/run-all-tasks.md`, and `logs/README.md` now encode the separation and the verification loop.
- **Applies to:** Future Zoro, Architect, and Ideas Hub coordination workflows
- **Confidence:** Verified by repository readback

## 2026-07-23 — Avoid recursive repository logging

- **Context:** Append-only repository activity logging
- **Learning:** Log-maintenance commits must not create new log obligations, or repository logging becomes recursive and noisy. The original repository event entry is sufficient; mailbox and report-only writes are also excluded unless they represent a material shared-system change.
- **Evidence:** `logs/README.md` and `architect/commands/run-all-tasks.md` explicitly define the non-recursion rule.
- **Applies to:** All future operational-log maintenance
- **Confidence:** Verified by repository readback
